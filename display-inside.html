The flow layout represents the default layout. It places inline elements in the writing direction of the used language, and block elements perpendicular to it.

table

The table layout is a composite layout that works exactly as the table elements do. To use it you need multiple other html elements with a specific display type to correspond to the different table elements.

Here is a list of table display types and their corresponding element:

  • table - table
  • table-row - tr
  • table-row-group - tbody
  • table-header-group - thead
  • table-footer-group - tfoot
  • table-column - col
  • table-column-group - colgroup
  • table-cell - td
  • table-caption - caption

flex

The flex layout is a unidirectional layout that provides tools for distributing space and aligning content.

The flex layout can be laid out in any direction (up, down, left or right). It allows you to specify the order of elements from css, control the way elements are aligned and how much they grow/shrink when the viewport shrinks.

For an easy reference of all the properties you can use in flex, I recommend this guide.

grid

The grid layout is a bidirectional layout, simillar to the table layout.

The grid layout allows you to define columns, rows and gutters. It also defines how much space columns will fill, both horizontally and vertically.

For an easy reference of all the properties you can use in grid, I recommend this guide.

ruby (experimental)

The ruby layout is a composite layout that behaves exactly like the ruby html element. To use it you need multiple child elements with specific display values, just like the table layout, each value linked to the ruby html elements.

Here is a list of ruby display types and their corresponding element:

  • ruby - ruby
  • ruby-base - rb
  • ruby-text - rt
  • ruby-base-container - rbc
  • ruby-text-container - rtc
